# astrbot_plugin_filter — 语言逻辑优化大师

🚿 元数据清洗 · 📝 工具名脱敏 · ✂️ LLM 语义分段 · 💬 多消息逐段发送 · 🛡️ 防 OOC

## 📋 这是什么？

当 AstrBot 接入 OneBot 协议向 QQ/微信输出消息时，LLM 回复常见三类问题：

1. **元数据泄漏**：`[{text=`、`, type=text}` 等 MCP/OneBot 结构符号暴露给用户
2. **工具痕迹**：bot 叙述内部操作过程、暴露工具函数名
3. **排版混乱**：长文本不分段、多个无关话题混在一条消息里

本插件在消息发送前最后一步（`on_decorating_result` 钩子）执行**六道管线**，一次性解决。

## 🔧 六道管线

```
LLM 生成回复 → ① 垃圾符号清洗 → ② 用户→昵称替换 → ③ 敏感信息过滤
                                   ↓
         用户收到干净分段消息 ← ⑥ 智能分段 ← ⑤ 工具名脱敏 ← ④ 删除叙述句
```

### ① 垃圾符号清洗

移除 OneBot/MCP 泄漏的元数据符号：`[{text=` `, type=text}` `}]` `[{` `}]`

### ② 用户 → 群昵称替换

从消息开头的 `@群昵称` 中提取实际昵称，将正文中的「用户」替换为真实称呼，让回复更自然、更像在跟真人对聊。

### ③ 敏感信息过滤

自动屏蔽系统路径（`/etc/` `/var/` `/AstrBot` 等）、Shell 命令片段、内网 IP、数据库连接/API key 等敏感字符串。

### ④ 过程叙述删除

识别并删除 bot 的内部操作叙述句。判定标准：句子中**同时出现**工具函数名 + 过程叙述关键词（如 "我先用" "让我" "执行操作" 等），缺一不可，避免误伤正常对话。

### ⑤ 工具名脱敏

将残存的工具函数名替换为自然语言，覆盖 40+ 条映射：

| 工具名 | 替换为 |
|--------|--------|
| `es_search` `rg_search` | 检索 / 文件检索 |
| `mikan_search` `bangumi_search` | 番剧源 / 番剧信息 |
| `ani-rss` `add_subscription` | 订阅系统 / 添加订阅 |
| `shell` `exec` `bash` | 终端 / 执行 |
| `read_file` `write_file` | 读取文件 / 写入文件 |
| …… | |

### ⑥ 智能分段（LLM 优先 + 规则降级）

| 模式 | 说明 |
|------|------|
| **LLM 语义分段** | 调用配置的轻量模型，按语义将文本拆成 3~4 段（寒暄引入 / 正文按话题分组 / 收尾关怀），同时检测无关话题混入 |
| **规则分段（fallback）** | LLM 不可用时，按已有空行 + 列表检测 + 句子均分进行纯规则分段 |

#### 多消息逐段发送

开启后，分段结果不再合并为一条长消息，而是**逐条发送**，段间自动插入 3~10 秒随机延迟，模拟真人打字/思考的自然节奏。

#### 防 OOC 机制

- **提示词硬约束**：只允许调整分段换行，严禁修改任何一个字、标点、语气词、emoji
- **中文字数校验**：LLM 输出与原文中文字数偏差超过 ±5% 即判定篡改，自动降级到规则分段
- bot 人设、语气、系统提示词完全不受影响

## 📦 安装

### 方式一：AstrBot 插件市场（推荐）

在 AstrBot 管理面板 → 插件市场中搜索 **语言逻辑优化大师**，点击安装。

### 方式二：手动安装

```bash
cd AstrBot/data/plugins
git clone https://github.com/Shtraiy/astrbot_plugin_filter.git
```

安装后重启 AstrBot 即可生效。

## ⚙️ 配置

插件提供 Web UI 配置面板（AstrBot 管理面板 → 插件 → 语言逻辑优化大师 → 配置）：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `enable_llm_segment` | 开关 | 关闭 | 启用 LLM 语义分段 |
| `llm_provider_id` | 下拉选择 | 空 | 选择用于分段的 LLM（推荐轻量模型） |
| `multi_message` | 开关 | 开启 | 逐条发送分段消息 |
| `delay_min` | 数字 | 3.0s | 消息间隔下限 |
| `delay_max` | 数字 | 10.0s | 消息间隔上限 |

## ⚙️ 兼容性

## 🐛 常见问题

### Q: 安装后没有生效？

- 确认 AstrBot 版本 >= 4.16
- 重启 AstrBot
- 在 AstrBot 日志中搜索 `[语言逻辑优化大师]` 确认插件已加载

### Q: 会不会误删正常消息？

不会。垃圾符号清洗只匹配特定元数据模式。过程叙述删除需要**工具名 + 过程关键词同时出现**才触发，正常对话不会同时满足。防 OOC 校验确保 LLM 分段不会篡改原文内容。

### Q: LLM 分段不工作？

1. 确保已在 AstrBot「系统配置 → LLM 供应商」中配置了至少一个 chat completion 类型的 provider
2. 在插件配置中选择该 provider
3. 打开 `enable_llm_segment` 开关
4. 检查日志中是否有 `[LLM分段]` 相关输出

### Q: 多消息模式会不会刷屏？

每条消息间隔 3~10 秒随机延迟，模拟真人打字节奏。如果不需要，可以关闭 `multi_message` 开关，所有段落将合并为一条消息发送。
